## 演示场景

### 场景一：使用模板快速开始（3分钟）

**目标**：展示模板系统的便捷性

**步骤**：
1. 登录后，点击左侧导航栏的 **"模板市场"**
2. 浏览可用模板，选择 **"行业研究报告生成"**
3. 点击 **"查看详情"**，查看模板结构
4. 点击 **"使用此模板"**，系统自动创建工作流
5. 跳转到工作流画布，查看自动生成的Agent和Task
6. 点击右上角 **"运行"** 按钮
7. 进入执行监控页面，观察：
   - Agent实时思考过程
   - 工具调用详情
   - 进度更新
   - Token消耗统计
8. 等待执行完成，查看生成的报告

**亮点**：
- 模板一键复用
- 可视化工作流设计
- 实时监控执行过程
- 自动生成专业报告

---

### 场景二：从零创建工作流（5分钟）

**目标**：展示工作流编辑器的强大功能

**步骤**：
1. 回到首页，点击 **"新建工作流"**
2. 输入工作流名称：**"竞品分析助手"**
3. 进入画布编辑器
4. **创建Agent节点**：
   - 从左侧面板拖拽 "Agent" 节点到画布
   - 双击节点，配置：
     - 名称：市场研究员
     - 角色：资深市场分析师
     - 模型：GPT-4o
     - 工具：web_search, file_read
   - 重复创建第二个Agent：报告写手

5. **创建Task节点**：
   - 拖拽 "Task" 节点到画布
   - 配置：
     - 名称：竞品信息收集
     - 描述：收集主要竞品的基本信息
     - 执行者：市场研究员
   - 创建第二个Task：报告撰写
     - 依赖：竞品信息收集

6. **连线**：
   - 将Task 1连接到Task 2，建立依赖关系

7. **校验**：
   - 点击 **"校验"** 按钮
   - 确认DAG结构正确

8. **运行**：
   - 点击 **"运行"** 开始执行
   - 观察执行过程

**亮点**：
- 拖拽式可视化编辑
- 灵活的Agent配置
- 依赖关系管理
- 实时DAG校验

### 场景三：实时监控和调试（3分钟）

6. **查看结果**：
   - 实时预览生成的内容
   - 支持Markdown渲染
   - 可以复制或导出

**亮点**：
- 完全透明的执行过程

- 专业结果展示

## 演示脚本

### 自动化演示脚本

如果你想进行自动化演示或测试，可以使用以下脚本：

#### 运行端到端测试

```bash
# 确保服务已启动
./docker-start.sh start

# 运行E2E测试
python tests/e2e_test.py
```

测试将自动验证：
- ✅ 用户注册和登录
- ✅ 工作流CRUD操作
- ✅ Agent和Task管理
- ✅ 执行引擎功能
- ✅ 边界条件处理

#### 使用API演示

```bash
# 1. 获取Token
TOKEN=$(curl -s -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"demo@fugue.com","password":"Demo123456"}' \
  | jq -r '.access_token')

echo "Token: $TOKEN"

# 2. 获取工作流列表
curl -s http://localhost:8000/api/v1/crews/ \
  -H "Authorization: Bearer $TOKEN" \
  | jq '.'

# 3. 获取模板列表
curl -s http://localhost:8000/api/v1/templates/ \
  -H "Authorization: Bearer $TOKEN" \
  | jq '.items | length'

# 4. 使用模板创建工作流
TEMPLATE_ID=$(curl -s http://localhost:8000/api/v1/templates/ \
  -H "Authorization: Bearer $TOKEN" \
  | jq -r '.items[0].id')

curl -s -X POST "http://localhost:8000/api/v1/templates/$TEMPLATE_ID/use" \
  -H "Authorization: Bearer $TOKEN" \
  | jq '.'

# 5. 启动执行
CREW_ID=$(curl -s http://localhost:8000/api/v1/crews/ \
  -H "Authorization: Bearer $TOKEN" \
  | jq -r '.[0].id')

curl -s -X POST http://localhost:8000/api/v1/executions/ \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"crew_id\":\"$CREW_ID\"}" \
  | jq '.'
```

## 常见演示问题

### Q1: 演示时服务启动失败怎么办？

**A:** 按以下步骤排查：

```bash
# 1. 检查Docker是否运行
docker --version
docker compose version

# 2. 查看详细日志
docker compose logs backend

# 3. 检查端口占用
# Windows
netstat -ano | findstr :8000
netstat -ano | findstr :3000

# Linux/macOS
lsof -i :8000
lsof -i :3000

# 4. 清理并重新启动
docker-start.sh cleanup
docker-start.sh start
```

### Q2: 演示时执行超时怎么办？

**A:** 执行时间取决于：
- LLM API响应速度
- 任务复杂度
- 网络状况

**建议**：
1. 使用简单的模板演示（如"代码审查助手"）
2. 确保网络连接稳定
3. 可以预先执行一次，缓存结果
4. 如果使用Mock模式，会更快

### Q3: 如何演示离线模式？

**A:** Fugue支持本地模型：

1. 安装Ollama
2. 下载模型：
   ```bash
   ollama pull llama3
   ```
3. 在Agent配置中选择：
   - Provider: `ollama`
   - Model: `llama3`
   - Base URL: `http://host.docker.internal:11434`

### Q4: 演示时如何展示错误处理？

**A:** 可以故意触发错误：

1. **无效配置**：
   - 创建一个Agent，不配置模型
   - 尝试运行，观察错误提示

2. **循环依赖**：
   - 创建A→B→A的依赖关系
   - 点击校验，观察错误提示

3. **API错误**：
   - 使用无效的API Key
   - 观察错误处理和提示

4. **网络错误**：
   - 断开网络
   - 观察重试机制

### Q5: 如何准备离线演示？

**A:** 提前准备好：

1. **启动服务**：
   ```bash
   ./docker-start.sh start
   ```

2. **预执行工作流**：
   - 运行几个工作流
   - 缓存执行结果

3. **准备截图**：
