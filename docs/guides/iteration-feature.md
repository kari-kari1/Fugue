# 执行结果迭代功能指南

本文档介绍如何使用 Fugue 的执行结果迭代功能，帮助你优化工作流执行结果。

## 功能概述

执行结果迭代功能允许用户在工作流执行完成后提供反馈，基于反馈让 AI 模型优化结果。通过多次迭代，你可以逐步获得更符合期望的输出。

### 核心特性

- **反馈驱动优化**：基于你的反馈进行结果改进
- **迭代历史保留**：所有历史版本都被保存，便于对比和回溯
- **灵活迭代模式**：支持增量优化和完全重新执行
- **Token 透明性**：每次迭代都会消耗 Token 费用

## 使用方法

### 1. 查看执行结果

工作流执行完成后，你可以通过以下方式查看结果：

**前端界面：**
- 执行完成后会自动显示结果详情页
- 结果页面包含完整的输出内容和元数据
- 可以通过"历史"标签查看所有版本

**API 方式：**
```bash
# 获取执行结果
curl -X GET "http://localhost:8000/api/executions/{execution_id}/results" \
  -H "Authorization: Bearer {api_token}"
```

### 2. 提交反馈

**在界面上提交：**

1. 打开执行结果详情页
2. 点击底部的"提交反馈"按钮
3. 在反馈表单中填写：
   - **反馈类型**：选择反馈类别（内容准确性、格式问题、遗漏信息等）
   - **具体描述**：详细说明需要改进的地方
   - **期望结果**：描述你期望的输出是什么样的

**API 方式：**
```bash
# 提交反馈
curl -X POST "http://localhost:8000/api/executions/{execution_id}/feedback" \
  -H "Authorization: Bearer {api_token}" \
  -H "Content-Type: application/json" \
  -d '{
    "feedback_type": "content_improvement",
    "description": "结果缺少关于 XX 的详细说明",
    "expected_output": "应该包含完整的 XX 分析"
  }'
```

### 3. 选择迭代模式

Fugue 提供两种迭代模式，根据你的需求选择：

#### 增量优化（Incremental）

适用于结果基本满意但需要微调的情况。

- **特点**：
  - 基于当前结果进行改进
  - 保留已完成的部分
  - 只针对反馈部分进行优化
  - Token 消耗相对较少

- **选择场景**：
  - 格式需要调整
  - 需要补充遗漏内容
  - 需要细化某些细节

**如何选择：**
1. 在提交反馈后，系统会提示选择迭代模式
2. 选择"增量优化"选项
3. 确认执行

#### 完全重新执行（Complete Rerun）

适用于当前结果存在重大问题或方向错误的情况。

- **特点**：
  - 基于新的反馈重新执行整个工作流
  - 不保留之前的结果
  - 从零开始生成新结果
  - Token 消耗较高

- **选择场景**：
  - 完全误解了任务意图
  - 需要改变整体方向
  - 当前结果无法修复

**如何选择：**
1. 在提交反馈后，系统会提示选择迭代模式
2. 选择"完全重新执行"选项
3. 确认执行

### 4. 查看迭代历史

所有迭代版本都会被保存，便于对比和选择最优结果。

**界面查看：**
1. 在执行结果详情页，点击"历史"标签
2. 按时间顺序显示所有版本
3. 每个版本显示：
   - 版本号（1, 2, 3...）
   - 创建时间
   - 反馈摘要
   - 结果预览
4. 点击任何版本查看完整内容
5. 可以选择任何版本作为当前结果

**API 查看：**
```bash
# 获取所有历史版本
curl -X GET "http://localhost:8000/api/executions/{execution_id}/history" \
  -H "Authorization: Bearer {api_token}"

# 获取特定版本
curl -X GET "http://localhost:8000/api/executions/{execution_id}/history/{version_id}" \
  -H "Authorization: Bearer {api_token}"
```

## 最佳实践

### 迭代前检查

1. **确认执行完成**：只有执行完成的工作流才能进行迭代
2. **分析当前结果**：找出具体需要改进的地方
3. **明确期望**：清晰描述期望的结果是什么

### 反馈撰写技巧

✅ **好的反馈示例：**
```
反馈类型：遗漏内容
描述：结果中没有提到成本分析，应该包括：
1. 预计开发成本
2. 运营成本估算
3. 投资回报分析

期望结果：完整的成本效益分析报告
```

❌ **不好的反馈示例：**
```
描述：结果不好，需要改进
```

### 迭代次数建议

- **轻度优化**：1-2 次迭代通常足够
- **复杂任务**：可能需要 3-5 次迭代
- **过度迭代**：超过 5 次通常表示需要重新思考任务需求

### 成本控制

每次迭代都会消耗 Token。建议：

1. **小批量反馈**：一次提供多个改进建议，而不是逐个提出
2. **增量模式优先**：除非必要，优先选择增量优化
3. **版本管理**：保留最佳版本，避免无谓的重新执行

## 常见问题

### Q: 迭代功能无法使用怎么办？

**可能原因及解决方法：**

1. 执行未完成
   - 检查执行状态是否为"已完成"
   - 如果是"运行中"，需要等待完成

2. Token 余额不足
   - 检查账户 Token 余额
   - 迭代前确保有足够的 Token

3. 权限问题
   - 确认你有该工作流的编辑权限
   - 联系管理员获取访问权限

### Q: 迭代后的结果会覆盖之前的结果吗？

**不会。** 所有历史版本都会被保留。你可以在迭代历史中查看和选择任何版本。

### Q: 如何取消正在进行的迭代？

在迭代过程中，你可以：

1. 点击"取消迭代"按钮
2. 之前的版本保持不变
3. 取消操作不会消耗额外的 Token（如果尚未开始执行）

### Q: 迭代的最佳时间是什么时候？

- **立即迭代**：执行完成后立即检查，及时发现问题
- **定期审查**：对于重要工作流，可以先整体审查后再迭代
- **避免拖延**：不要保留过多未处理的执行结果

### Q: 可以同时对多个结果进行迭代吗？

可以。每个执行结果独立维护自己的迭代历史，可以并行处理多个迭代任务。

## API 参考

### 提交反馈接口

```http
POST /api/executions/{execution_id}/feedback

Request Body:
{
  "feedback_type": "string",  // 反馈类型
  "description": "string",     // 详细描述
  "expected_output": "string"  // 期望结果（可选）
}

Response:
{
  "success": true,
  "feedback_id": "uuid",
  "message": "反馈已接收，等待处理"
}
```

### 启动迭代接口

```http
POST /api/executions/{execution_id}/iterate

Request Body:
{
  "mode": "incremental|complete",  // 迭代模式
  "feedback_id": "uuid"           // 对应的反馈 ID
}

Response:
{
  "success": true,
  "iteration_id": "uuid",
  "status": "started",
  "estimated_time": "seconds"
}
```

### 获取迭代历史接口

```http
GET /api/executions/{execution_id}/history

Response:
{
  "versions": [
    {
      "id": "uuid",
      "version": 1,
      "created_at": "timestamp",
      "feedback_summary": "string",
      "status": "completed|in_progress"
    }
  ]
}
```

## 故障排除

### 常见错误

**错误 1：403 Forbidden**
- 原因：没有访问权限
- 解决：检查工作流的所有者或权限设置

**错误 2：404 Not Found**
- 原因：执行 ID 不存在
- 解决：确认执行 ID 是否正确，执行是否已删除

**错误 3：409 Conflict**
- 原因：有其他迭代正在运行
- 解决：等待当前迭代完成或取消后再启动新的

如果遇到问题：

4. 联系开发团队

## 总结

执行结果迭代功能是 Fugue 中的核心特性之一，通过迭代可以显著提升输出质量。关键要点：

- 只有执行完成后才能迭代
- 选择合适的迭代模式（增量或完全重新执行）
- 提供清晰具体的反馈
- 善用迭代历史管理版本
- 注意 Token 成本控制

通过合理使用迭代功能，你可以快速获得符合期望的高质量输出。
